Designing Effective Prompts

Learn how to build prompts that get better results from AI. Good prompts aren't written once—they're refined through conversation and iteration.

Core Principles

  • Start with context before giving commands
  • Be specific about your goal, not the implementation
  • Use examples to clarify your expectations
  • Iterate based on the AI's output
  • Build complexity gradually through conversation

Side-by-Side Comparison

Here's how context and specificity transform a prompt from vague to effective:

Less Effective
"Create a function to sort data"

Why it's less effective: No context about what kind of data, how it should be sorted, or what edge cases to handle. AI has to guess your intent.

More Effective
"I'm building a dashboard that displays user analytics. I need to sort an array of user objects by signup date, newest first. The array might be empty, and dates are stored as ISO strings. Can you create a sorting function that handles these cases?"

Why it's better: Provides context (dashboard), specific requirements (newest first, ISO strings), edge cases (empty array), and clear expectations.

Using AI to Build Your Prompt

One of the most powerful techniques is using AI to help you craft better prompts. Start with a rough idea of what you need, then ask AI to help you refine it:

Example workflow:

  1. Share your rough idea with AI
  2. Ask AI what additional context would be helpful
  3. Refine the prompt together through conversation
  4. Arrive at a clear, effective prompt for your task

This iterative approach means you don't need to know the "perfect" way to prompt upfront. The orchestration model applies to prompt design itself.
